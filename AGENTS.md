# Agent 工作指引

## 必讀文件

在分析、修改或產生程式碼之前，必須依序閱讀：

1. `SPEC.md`
2. `PLAN.md`
3. 與任務相關的原始碼及測試

在確認適用的需求與目前里程碑之前，不得開始實作。

## 事實來源

* `SPEC.md` 定義必要行為、介面、限制與驗收條件。
* `PLAN.md` 定義實作里程碑、工作順序與驗證程序。
* 當既有程式碼與 `SPEC.md` 衝突時，不得直接假設既有程式碼正確。

## 工作流程

1. 在 `SPEC.md` 中確認本次工作的需求。
2. 在 `PLAN.md` 中確認目前所屬里程碑。
3. 檢查相關程式碼與測試。
4. 實作能完整滿足需求的最小變更。
5. 執行該里程碑定義的驗證命令與相關測試。
6. 回報以下內容：
   * 變更的檔案
   * 已實作的行為
   * 已執行的驗證
   * 尚未解決的問題

## 限制

* 不得加入 `SPEC.md` 未描述的功能。
* 不得在未說明的情況下變更公開介面。
* 驗證未通過時，不得將里程碑標記為完成。
* 不得僅為了讓驗證通過而移除失敗的測試。
* 文件必須與實作保持同步。
